01

我第一次真正理解 shader 是什麼的時候，不是在讀文件，而是在盯著一個球體的高光，突然想到： 這個反光的形狀，其實是某個人在某個地方寫下的一行數學。 那一刻，技術和美學對我來說變成了同一件事。

從數學到感知

Shader 本質上是一組對每個像素執行的函數。輸入是位置、法向量、光源方向， 輸出是一個顏色值。聽起來乾燥，但這個過程裡藏著所有視覺美感的底層邏輯—— 我們看到的每一個「好看」，都是某種數學結構的投影。

Phong 模型把光分成 ambient、diffuse、specular 三個部分。 這不只是物理近似，也是一種對「什麼讓東西看起來有質感」的哲學選擇。 你決定高光的銳利度，你決定環境光的溫度，你在選擇一種觀看世界的方式。

"每一行 GLSL 都是一個關於光如何運作的假設。而好的渲染，是選對了假設。"

— 寫第一個自定義 shader 時的筆記

Shader 是一種語言

我後來開始把 shader 想成語言，而不是程式。它有語法，但更重要的是它有語意—— 同樣的光照模型，用不同的參數組合，說的是完全不同的故事。 冷色調的 specular 說的是金屬，暖色調漫反射說的是皮膚。

這讓 Technical Artist 這個工作對我來說有了新的意義： 我不只是在調參數，我在翻譯——把藝術家腦海中的畫面， 翻譯成渲染引擎能理解的語言。

光的方程式

學 GLSL 的過程裡，我遇到最多的問題不是語法，而是「為什麼這樣看起來對」。 這個問題沒有標準答案，因為它涉及人類視覺系統的偏好、文化對美的定義、 還有你想讓觀看者感受到什麼。

GLSL — Simple Phong Specular
// 計算高光強度
vec3 viewDir = normalize(camPos - fragPos);
vec3 reflectDir = reflect(-lightDir, normal);
float spec = pow(
  max(dot(viewDir, reflectDir), 0.0),
  32.0  // shininess — 越高越銳利
);
vec3 specular = spec * lightColor * 0.5;

那個 32.0 是什麼？ 是一個選擇。是說「這個材質的高光這麼銳」的一個決定。 數字本身沒有意義，意義在你為什麼選它。

技術背後的美學選擇

我現在做 Technical Artist 的工作，很多時候就是在處理這種「技術和美學交界處的決定」。 工程師問的是「這樣實作對嗎」，藝術家問的是「這樣看起來對嗎」， 而我需要讓兩個問題的答案同時是 yes。

這不容易，但我越來越覺得這是最有趣的部分—— 在兩種語言的邊界上工作，學著翻譯彼此都能理解的東西。 Shader 教會我的，不只是怎麼寫光，而是怎麼在精確和感性之間找到位置。

"技術是手段，感知是目的。好的 shader 讓人忘記它的存在。"
